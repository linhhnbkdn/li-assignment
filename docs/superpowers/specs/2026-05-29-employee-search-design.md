# Employee Search Directory — Design Spec

**Date:** 2026-05-29
**Scope:** FastAPI microservice — search/filter API only (no CRUD)

---

## 1. Requirements Summary

| Requirement | Decision |
|---|---|
| Framework | FastAPI (Python) |
| Database | SQLite + SQLAlchemy 2.0 ORM |
| Migrations | Alembic |
| Auth | JWT Bearer token (per-org secret) |
| Rate limiting | Custom sliding-window, per user, no external lib |
| Column visibility | Per-org config stored in `org_column_configs` table |
| Pagination | Cursor-based (`after_id`) — no OFFSET |
| Full-text search | SQLite FTS5 virtual table |
| Containerized | Dockerfile + docker-compose.yml |
| API format | OpenAPI (auto-generated by FastAPI) |
| Tests | pytest — unit tests (services/repos) + integration (TestClient) |

---

## 2. Clean Architecture Layers

Strict downward dependency — upper layers never import from lower layers' peers.

```
HTTP (api/)           → schemas, services
  ↓
Services (services/)  → repos, domain models
  ↓
Repos (repos/)        → ORM models, DB session
  ↓
DB (core/db.py)       → SQLAlchemy engine

Cross-cutting (core/): auth, rate_limiter, config, logging
  → imported by any layer, never import from api/services/repos
```

| Column | Type | Notes |
|---|---|---|

## 6. API Contract

```
GET /api/v1/employees/search
Authorization: Bearer <jwt>

Query params (all optional):
  q           string    Full-text search on name + email (FTS5)
  department  string    Exact match filter
  location    string    Exact match filter
  position    string    Exact match filter
  limit       int       Default 20, max 100
  after_id    string    Cursor UUID for next page

Response 200:

  "items": [
```

## 7. Dependency Injection Chain

```python
# deps.py
def get_db() -> Generator[Session, None, None]:
    db = SessionLocal()
    try:
        yield db
    finally:
        db.close()

def get_current_user(
    token: str = Depends(oauth2_scheme),
    db: Session = Depends(get_db),
) -> User: ...  # decodes JWT, returns User ORM object

def require_rate_limit(
    user: User = Depends(get_current_user),
) -> None: ...  # raises HTTP 429 if exceeded

# Route
@router.get("/employees/search", response_model=SearchResponse)
def search(
    params: Annotated[SearchParams, Depends()],
    user:   User = Depends(get_current_user),
    _:      None = Depends(require_rate_limit),
    db:     Session = Depends(get_db),
) -> SearchResponse: ...
```

## 8. Rate Limiter

Custom sliding-window, in-memory, thread-safe:

```python
class SlidingWindowRateLimiter:
    def __init__(self, limit: int = 30, window_seconds: int = 60) -> None:
        self._buckets: dict[str, deque[float]] = defaultdict(deque)
        self._lock = threading.Lock()

    def check(self, user_id: str) -> None:
        now = time.monotonic()
        with self._lock:
            bucket = self._buckets[user_id]
            while bucket and bucket[0] < now - self._window:
                bucket.popleft()
            if len(bucket) >= self._limit:
                retry_after = int(self._window - (now - bucket[0]))
                raise RateLimitExceeded(retry_after=retry_after)
            bucket.append(now)
```

Global singleton instantiated in `main.py`, passed via `app.state.rate_limiter`.

## 9. Column Masking (no data leaks)

```python
# services/search_service.py
def _mask_employee(self, employee: Employee, allowed_cols: list[str]) -> dict:
    base = {"id": employee.id, "name": employee.name}  # always returned
    for col in allowed_cols:
        if col != "name" and hasattr(employee, col):
            base[col] = getattr(employee, col)
    return base
```

Column config is loaded once per request from `OrgRepo`, cached on the `User` object within the DI chain. Masking happens before Pydantic serialization.

## 11. Testing Strategy

| Test file | Type | What it covers |
|---|---|---|
| `test_search_service.py` | Unit | filter logic, column masking, cursor pagination |
| `test_rate_limiter.py` | Unit | sliding window, 429 threshold, window reset |
| `test_search_route.py` | Integration | full HTTP flow, JWT validation, 401/422/429 |
| `conftest.py` | Fixtures | in-memory SQLite, seeded orgs/users, JWT generation |

Services and repos are tested by injecting a real in-memory SQLite session — no mocking of DB.
